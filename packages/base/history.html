4calc.valueOf(); // 2
5calc.mut(2);
6calc.valueOf(); // 4
7
8// getCurrValue 不会真实计算结果
9const calc2 = new Calculator(1);
10calc2.add(1);
11calc2.getCurrValue(); // 2;
12calc2.mut(2);
13calc2.valueOf(); // 3

#v0.4.0

新增

  • (F) curry
  • (F) compose
  • (F) pipe

警告

compose 和 pipe 类型存在缺陷，只能判断最后输入的函数是否满足条件，不能判断中间的函数 也就是说从前往后传入函数可以正确判断类型，但是如果是在中间插入函数就不行，而且对于柯理化后的函数，只能判断第一个参数的类型
